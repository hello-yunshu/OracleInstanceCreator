## 三、代码优化

### 高优先级 Bug 修复

1. **`infrastructure-deployment.yml` 第 123-124 行：条件表达式逻辑错误**
   - 现状：`ENABLE_ADAPTIVE_SCHEDULING: ${{ inputs.adaptive_scheduling && 'true' || 'true' }}`
   - 问题：无论输入如何都返回 `'true'`，`|| 'true'` 导致条件永远为真
   - 修复：改为 `${{ inputs.adaptive_scheduling && 'true' || 'false' }}`
   - 同理修复 `ENABLE_REGION_OPTIMIZATION`

2. **`launch-parallel.sh` 第 677 行：bash 不支持三元运算符**
   - 现状：`$(((a1_duration + e2_duration) > 0 ? (a1_duration + e2_duration) * 100 / elapsed : 0))`
   - 问题：bash 算术不支持 `? :` 三元语法
   - 修复：改用 if/else 或 `((sum > 0)) && echo $((sum * 100 / elapsed)) || echo 0`

3. **`circuit-breaker.sh` 第 88 行：macOS 兼容性问题**
   - 现状：`date -d "$last_failure_time" +%s` 在 macOS 上不工作
   - 修复：添加 macOS 兼容处理，使用 `python3` 或 `perl` 做时间转换

### 中优先级优化

4. **退出码常量命名不一致**
   - `constants.sh` 使用 `EXIT_SUCCESS`、`EXIT_GENERAL_ERROR` 等
   - `utils.sh` 使用 `OCI_EXIT_SUCCESS`、`OCI_EXIT_GENERAL_ERROR` 等
   - 修复：统一为 `OCI_EXIT_*` 命名（更明确），删除 `utils.sh` 中的重复定义，全部使用 `constants.sh` 中的常量

5. **`launch-instance.sh` 重复代码提取**
   - 第 347-381 行（首次成功）和第 479-512 行（重试成功）逻辑几乎相同
   - 优化：提取为 `handle_launch_success()` 函数，参数区分首次/重试

6. **`oci_cmd_debug` 硬编码超时值**
   - 现状：`--connection-timeout "5"` 和 `--read-timeout "15"` 硬编码
   - 优化：使用 `constants.sh` 中的 `OCI_CONNECTION_TIMEOUT_SECONDS` 和 `OCI_READ_TIMEOUT_SECONDS`

7. **`state-manager.sh` 性能优化**
   - 大量 debug 日志字符串在非 debug 模式下也会拼接
   - 优化：在拼接前检查 `INTERNAL_DEBUG` 变量

## 五、中文化原则

1. **注释**：所有英文注释翻译为中文
2. **log 消息**：所有 `log_info`、`log_warning`、`log_error`、`log_debug`、`log_success` 的消息文本翻译为中文
3. **通知消息**：Telegram 通知内容翻译为中文
4. **变量名/函数名**：保持英文不变（代码规范）
5. **OCI CLI 参数**：保持英文不变（API 规范）
6. **JSON key**：保持英文不变
7. **环境变量名**：保持英文不变
8. **错误类型字符串**：如 `"CAPACITY"`、`"RATE_LIMIT"` 等保持英文（用于 case 匹配）
9. **代码注释风格**：使用 `# 中文注释` 格式，不加多余装饰
